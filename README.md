# SOC Analyst Labs

This repository contains hands-on labs focused on Security Operations Center (SOC) investigation and detection techniques.

The labs demonstrate how a security analyst can investigate system activity, create detection logic, and analyze incidents using logs and monitoring tools.

The exercises are designed to be lightweight and reproducible on a personal laptop, with controlled VM usage depending on the type of lab.

## Repository Structure

### windows-investigation

Labs focused on investigating Windows endpoint activity using built-in tools.
These exercises demonstrate the ability to analyze Windows Event Logs and identify relevant security events.

Examples of activities in these labs include:

+ locating relevant Windows Event IDs
+ analyzing service installation events (e.g., Event ID 7045)
+ verifying whether software installation is legitimate
+ distinguishing between normal system activity and suspicious behavior

Lab characteristics:

+ 1 VM active
+ Windows endpoint only
+ No attack simulation
+ These labs focus on log familiarity and endpoint investigation skills.

### linux-investigation

Labs focused on investigating Linux system logs using built-in tools.
These exercises demonstrate the ability to navigate and analyze Linux logs to identify relevant system activity.

Examples of activities include:

+ reviewing authentication logs
+ identifying login activity
+ investigating system events and processes
+ distinguishing between expected and suspicious behavior

Lab characteristics:

+ 1 VM active
+ Linux endpoint only
+ No attack simulation
+ These labs focus on basic Linux log analysis and investigation workflows.

### siem-detection

Labs focused on building SIEM detection logic.
These exercises demonstrate how logs from endpoints can be ingested into a SIEM platform and used to create detection rules.

Examples of activities include:

+ writing detection queries
+ identifying suspicious patterns in log data
+ developing basic alert logic

Lab characteristics:

+ 2 VMs active at the same time
+ Linux (endpoint) + SIEM platform or Windows (endpoint) + SIEM platform
+ Endpoint logs forwarded to SIEM
+ No attack simulation
+ These labs focus on detection engineering and alert creation.

### incident-response-scenarios

Labs focused on incident investigation after detection.
These scenarios simulate suspicious activity within the lab environment so that investigation techniques can be demonstrated.

Examples of activities include:

+ analyzing alerts generated by detection rules
+ investigating suspicious activity across multiple systems
+ identifying attacker behavior using logs and monitoring tools

Lab characteristics:

+ 3 VMs active
+ SIEM + endpoint + Kali attacker
+ attack simulation performed inside the lab
+ These labs focus on incident triage and investigation workflows.

Lab Design Philosophy

+ The labs are structured to progressively demonstrate different SOC skill sets:
+ Endpoint log investigation
+ Detection rule creation
+ Incident investigation after detection
+ The environment is intentionally designed to run on a limited hardware setup, ensuring that labs remain practical to reproduce on a personal laptop.
